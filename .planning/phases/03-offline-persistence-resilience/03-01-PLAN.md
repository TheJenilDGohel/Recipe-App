    - When `getMealDetails(id)` is called:
      1. Check local cache.
      2. If cached and < 24h old, return cached meal.
      3. Otherwise, fetch from remote, save to cache, and return.
    - When `searchMeals(query)` is called:
      1. Fetch from remote (always fresh for search).
      2. Save results to cache for individual lookup resilience.
  </behavior>
